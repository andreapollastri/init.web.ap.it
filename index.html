How it works

Three steps from zero to running.

No Homebrew packages, no global npm installs, no manual config files. Just Docker Desktop and one command.

01
Run the installer
Paste the install command in your terminal. The script downloads and runs itself — no global install needed.
curl https://init.web.ap.it/bin/go.sh > go.sh && sh go.sh
02
Name your project
The script will ask for a project name. That name becomes your local subdomain — no hosts file edits, no config needed.
Project name: my-awesome-blog
03
Open your browser
The script prints all your endpoints — app URL, admin panel, Mailpit, MinIO, database credentials. Everything is ready.
→ https://my-awesome-blog.127001.it
Powered by 127001.it

A real domain for your localhost.

init.web.ap.it uses 127001.it — a free wildcard DNS service — to give every project its own subdomain that resolves to 127.0.0.1. No /etc/hosts edits. No sudo. No setup.

Type Name Value Note
A *.127001.it 127.0.0.1 ; wildcard — every subdomain
https://my-blog.127001.it → resolves to 127.0.0.1 · your local app
https://company-crm.127001.it → resolves to 127.0.0.1 · your local app
https://saas-app.127001.it → resolves to 127.0.0.1 · your local app

Take it to production

Deploy Laravel to any VPS.
Without the DevOps headache.

Once your app is ready locally, deploy it to production with cipi.sh — an open-source CLI built exclusively for Laravel. One command installs a complete production stack on any Ubuntu VPS.

Full app isolation Own Linux user, PHP-FPM pool & database per application.

Security out of the box Let's Encrypt SSL, Fail2ban, UFW firewall — all automated.

Multi-PHP support PHP 7.4 → 8.5. Queue workers, S3 backups, webhooks included.
